IMS Priorities

The Ranger IMS associates a priority level with each incident. This can provide a way to identify incidents that need greater attention or can be safely ignored for a while, which may be useful for managing Operator and Khaki workload when there are many open incidents to manage.

In order for priorities to be useful, however, it's important that they be used consistantly. To that end, the list below itemizes the priority levels and how they should be utilized:

High
We need to deal with this ASAP.
Normal
Default priority. Handle this as soon as resources allow.
Low
This is not resolved, but does not need attention right now.
